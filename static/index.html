Self Deploy Modes (Included in Repo)

The Space repo includes standalone local hosting via self_deploy with three runtime topologies.

Single Container

Proxy + capsule in one service. Best default for local operators.

cd Champion_Council/self_deploy docker compose -f docker-compose.yml up --build

Split Proxy and Capsule

Separate services for lighter proxy scaling and independent capsule lifecycle control.

cd Champion_Council/self_deploy docker compose -f docker-compose.split.yml up --build

Proxy to Remote Capsule

Use local panel + proxy against any remote MCP capsule endpoint.

cd Champion_Council/self_deploy MCP_BASE_URL=http://your-remote-capsule:8766 docker compose -f docker-compose.remote.yml up --build
Local setup docs: self_deploy in this Space repo. Fork this Space if you need private persistence and isolated secrets.

Fork for Personal Use

Create your own private copy and run with your own secrets and policies.

  • Duplicate this Space from tostido/Champion_Council.
  • Add secret HF_TOKEN in your Space settings.
  • Set optional vars: SPACE_AUTHOR_NAME, SPACE_ID.
  • Factory rebuild after secrets are configured.
HF_TOKEN=hf_xxx SPACE_AUTHOR_NAME=your_hf_username SPACE_ID=your_space_name
