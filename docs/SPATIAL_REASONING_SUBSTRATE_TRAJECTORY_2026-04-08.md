## Main Honest Seams

### 1. `supportingFeet` is still the real entry filter

The current support/load pipeline is still feet-first.

That means:

- non-foot contact families exist conceptually
- but downstream support/load truth is still constrained by foot-only seeding

### 2. Support-surface truth still has two flavors

The environment runtime has authored support surfaces and support records.
The workbench still leans on the local workbench floor.

That is acceptable for current builder work, but it is a real seam that must close before environment-mounted support interactions become first-class.

### 3. Text theater still consumes a reduced contract

The text lane is much better than before, but it still does not consume one final browser-authored post-stage display contract.

That remains the long-term parity shortcut.

### 4. Invisible mechanics are still more mature than visible mechanics

The load/support substrate is stronger than its current visual manifestation.

The next visible lane should remain:

- bone-first
- carpenter-level first
- scaffold secondary

## Immediate Trajectory

1. close remaining contract lies in web/text/shared-state/mechanics surfaces
2. continue the mechanics substrate return:
   - keep settle explicit
   - unify support truth further
   - generalize from `supportingFeet` toward `supportingContacts`
3. make the existing mechanics visible:
   - bone-native diagnostics
   - innerlay/load visualization
4. expand environment/text contract coverage where still partial
5. only then wire Dreamer onto the same truthful substrate

## Dreamer-Specific Next Step

Do not use Dreamer as a separate speculative intelligence layer.

First update:

- feed Dreamer the same shared environment/workbench/mechanics truth already exposed to other surfaces

Initial observation schema should stay modest:

- current mode / scope
- support surface kind
- support count
- active supporting set
- projected COM
- support polygon summary
- stability risk
- contact states
- current command / action
- assertion result
- compact environment context around the focused runtime

That turns Dreamer from "active but blind" into "active and grounded."

The next implementation pass should preserve that unity instead of multiplying side truths.
