## Purpose

Capture `Atrai` as the operator-vessel frame for the kinetic-architecture lane without confusing that naming overlay for a runtime contract.

`Atrai` is not the mechanism itself.
`Atrai` is the vessel-frame through which the operator, motion system, service shell, and civic outputs are held together as one readable stack.

## Bottom Line

Use `Atrai` when the need is:

- operator-facing program identity
- vessel/cell framing
- one-cell-to-constellation scope language
- mythic charge that still docks to real mechanics

Do not use `Atrai` to replace:

- runtime contracts
- primitive names
- safety envelopes
- charge / catch / return accounting

Short law:

- runtime names win every collision
- `Atrai` names the carried vessel-frame over them

## Three Registers

| Register | Short read |
|---|---|
| `caveman` | One rider, one big carried thing, one safe shell, one path through danger. |
| `operator` | One operator-vessel cell that extends sense, signal, commit, and service as one Vitruvian reach envelope. |
| `engineer` | One bounded kinetic utility platform composed of operator station, motion core, service bands, and public-output lanes. |

## Working Composition

Name composition held in continuity:

- `Atreyu` heritage: rider and mount as one operational pair
- `Atari` heritage: computing/game lineage

Mythic under-image:

- `Falkor` remains a cultural template, not a claim about runtime form

## Scope Ladder

### 1. Cell

First honest scope:

- one vessel cell
- one motion core
- one service shell
- one public-output relation

### 2. Facility

Second scope:

- one `brohandoskygivingfacility`
- one civic-protection translation path
- one bounded service architecture

### 3. Constellation

Later scope:

- multiple cells
- linked facilities
- carrier-fortress or orbital lattice interpretation

The ladder matters because the program should not pretend it starts at the top.

## Mechanical Docking

`Atrai` docks to real mechanism only through bounded parts:

- `primary_span`
- `spine_pivot`
- `carrier_arm`
- `relay_segment`
- `panel_leaf`
- `counterweight_carriage`
- `transfer_floor`
- `clean_band`
- `dirty_band`

If the vessel frame cannot name those lower parts, it is drifting into pure ornament.

## Visual Diagnostic

When someone says "show me Atrai," the drawing should reveal all of these:

1. operator position
2. motion core
3. charge source
4. catch / return path
5. service shell
6. outward value path

If the picture shows only dramatic silhouette, it is not ready.

## Guardrails

1. `Atrai` is an overlay, not a physics proof.
2. It does not grant perpetual motion.
3. It does not erase the distinction between public perception value and lower-band service truth.
4. It must always dock to a real charge bank, a real catch path, and a real utility sink.
