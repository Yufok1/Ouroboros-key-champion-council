  Desktop Companion Delivery Lane — SPEC DRAFT (2026-03-29)

  New delivery form for character products: desktop-native 3D agent companion. Same character product,
  same GLB, same manifest, same command surface, same animation contract — different runtime host.

  Two product orientations:
  - Orientation A (Environment-Primary): 3D scene is the main resource, agent is actor/performer within it
  - Orientation B (Agent-Primary): 3D character IS the interface, summons environment fragments as needed

  Desktop Perception Surface: Agent perceives and reacts to the user's desktop interaction context —
  window focus, app launches, cursor position, user activity tempo. Permission tiers Level 0–4
  (none → window awareness → app awareness → content awareness → action capability).

  Stack: Electron + Three.js (transparent frameless window, click-through on transparent pixels).
  Desktop physics: window title bars as platforms, taskbar as ground, screen edges as walls, cursor
  as interactive entity. Character renderer adapted from theater (fixed camera, simplified lighting,
  transparent shadow catcher, <20k triangles).

  Phases: (1) Current v133 lane completes first — animation system IS the foundation. (2) Desktop
  shell + physics. (3) Desktop intelligence (perception, cursor interaction). (4) Desktop agent
  (voice, environment summoning, grapple/swing). (5) Desktop products (packaging, marketplace).

  Full spec: docs/DESKTOP_COMPANION_ARCHITECTURE_SPEC_2026-03-29.md
