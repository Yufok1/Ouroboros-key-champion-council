# Hold Door Raid Gamification Spec 2026-04-19

## Purpose

Turn the holder-first creator-pool doctrine into visible runtime packets without collapsing into hype slop.

This spec keeps four truths separate:

- `camp` = holder patience and stabilization
- `raid` = verified contribution and pressure work
- `settlement` = frozen epoch money truth
- `report` = the human-readable delivery artifact

## Runtime Packets

### 1. `holder_snapshot_packet`

Role:

- body/camp lane
- holder qualification seed
- anti-snipe posture

Core ideas:

- `camp_label = campers`
- `qualification_mode = age_weighted_sqrt_hold`
- `qualification_window = 7d_rolling`
- `anti_snipe_mode = age_integrity_discount`

It must reward:

- patience
- integrity
- stabilization

It must not reward:

- bag stuffing
- sniper entry
- pure concentration

### 2. `raid_contribution_packet`

Role:

- Platinum DKP lane
- competitive specialist grammar

Core ideas:

- `raid_label = guerrilla_specialists`
- `evidence_mode = manual_verification`
- `scoring_formula = verified_contribution_plus_negentropy_minus_entropy`
- role families:
  - holder
  - builder
  - operator
  - scout
  - amplifier
  - stabilizer

It must reward:

- value-positive contribution
- truth-telling
- negentropy work

It must not reward:

- churn
- spam
- fake motion
- wash behavior

### 3. `settlement_epoch_packet`

Role:

- frozen epoch settlement authority

Core ideas:

- `settlement_asset = SOL`
- `settlement_style = snapshot_then_manual_release`
- `release_governance = hold_door_epoch`
- `circuit_breaker_mode = hold_then_implode`
- `reserve_cover_target_epochs = 4`

Macro split:

- `Body` = 45%
- `Raid` = 20%
- `Shield` = 20%
- `Forge` = 15%

### 3.1 `threat_bounty_packet`

Role:

- shield-side threat jackpot
- survival bounty lane

Core ideas:

- `jackpot_mode = verified_threat_fast_release`
- `adjacent_reward_mode = finder_cut_plus_double_to_jackpot`
- `verification_mode = manual_verification_required`
- `critical_trigger_mode = hold_then_fast_release`
- `panic_farming_penalty = enabled`

Payout grammar:

- critical verified threat = full jackpot
- adjacent real threat = finder cut only
- then double the finder cut and add that amount back into the jackpot pool

This lane must reward:

- early real threat discovery
- evidence
- containment value

This lane must not reward:

- rumor spam
- panic farming
- self-created threats
- low-signal drama

## Game Clock Ladder

The system should stay alive on multiple clocks.

### Hourly

- pulse
- raid pressure
- chatter heat
- field readiness

### Daily

- qualification checkpoint
- holder snapshot update
- raid evidence checkpoint

### Weekly

- macro settlement
- body/raid/shield/forge ledger update
- Hold Door release review

### Monthly

- policy review
- role tuning
- reserve health review

### Quarterly

- seasonal championship
- major reset/reclassification window

## Delivery Artifact

### 4. `hold_door_raid_report_packet`

Display name:

- `Hold Door Raid Report`

Role:

- shared certificate culture plus individualized receipt truth

Tone:

- dead serious
- ridiculous
- wholesome
- bureaucratically warm

Recommended embodiment:

- tiny block courier
- sharp suit
- minimal geometry
- no distracting detail

## Failure Watches

These should be watched explicitly now, not later:

- concentration spike
- sybil or snipe pressure
- reserve drain
- jackpot farm behavior
- proof gap
- manual release bypass

If these trip:

1. `HOLD`
2. freeze the release lane
3. recorroborate proof surfaces
4. `implode` only if the lane cannot be trusted

## Joke Discipline

The lane can carry absurd delivery language, but the logic underneath must stay literal.

Example allowed tone:

- `policies`
- `politickin`
- `picturin`
- `polic... watch the lice`

Meaning:

- silly public line
- strict verification underneath

## What This Must Not Become

- not a casino leaderboard
- not whale worship
- not spam-for-rewards
- not instant payout addiction
- not opaque operator favoritism
- not yield theater without proof

## Bottom Line

The game should create identity and energy, but the settlement must stay boring:

- lively clocks
- frozen epochs
- visible proof
- restrained release

That is how the game survives success without destroying itself later.
