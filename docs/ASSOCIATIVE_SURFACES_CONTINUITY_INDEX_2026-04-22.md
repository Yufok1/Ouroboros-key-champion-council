## Purpose

Expose the actual associative surfaces through the continuity lane so a reset or reacclimation lands on the real carrying seams instead of transcript folklore.

This is not a new authority plane.

It is the canonical index of:

- which surfaces carry truth
- which surfaces carry orientation
- which surfaces carry archive/planning posture
- which surfaces consume that posture downstream

## Bottom Line

The clean reading is:

- associative surfaces are the shared named faces that keep the active objective, subject, pivot, docs, continuity, support, and visible observation aligned
- some of them are state-machine-like
- some of them are packets
- some of them are downstream consumers
- they are not all the same thing

The rule:

- one truth spine
- one evidence order
- one derived orienting crane
- many bounded associative surfaces over that same truth

## Authority Order

1. live runtime truth
2. live theater and capture corroboration
3. `blackboard.working_set.query_thread`
4. `output_state`
5. scoped reports / contracts
6. docs and continuity archive faces

No associative surface below live truth gets to overrule live truth.

## Guardrails

1. `query_thread` and `output_state` remain the core carrying surfaces.
2. `Tinkerbell` points; `Pan` measures; they do not merge into one hidden god-system.
3. `Dreamer`, HOLD, Coquina, palette families, and weather families must consume the carrying surfaces instead of bypassing them.
4. Downstream embodiment lanes may amplify truth, but must not overwrite it.
5. Docs and continuity packets stay bounded planning/archive faces, not replacement runtime.
6. `workspace_packet` must make the current server-visible work lane explicit without pretending the host filesystem is already mirrored into shared state.
7. `entry_gate` must keep user claims, transcript residue, and single-surface reads from being treated as sufficient evidence by themselves.
8. `misunderstanding_box` must isolate active incomprehension without letting archive, pointer, or proposer lanes hallucinate the seam closed.

## Spatial Intuition / Containment Model

When the operator speaks in terms of rooms, service floors, onions, furnaces, dumbwaiters, counterweights, or Rubik-style rotation, map the continuity stack like this:

- core / furnace = live runtime truth plus active seam pressure
- upper clean occupancy = live theater, `blackboard.working_set.query_thread`, and `output_state`
- lower service / recovery bands = `continuity_packet`, `docs_packet`, archive posture, stale carried state, and misunderstanding boxes
- transfer floor / service lift = `continuity_restore`, paired-state alignment, `trajectory_correlator`, `continuity_cue`, and scoped reports
- pointer lane = `Tinkerbell`
- measurement lane = `Pan`
- bounded proposer/scorer lane = `Dreamer`

Use this model to preserve safe footing:

- keep the upper room readable and habitable
- push stale, dirty, or unresolved material into bounded service bands
- let transfer surfaces move that material deliberately instead of smearing it across the live room
- only let cleaned or corroborated material return upward

Rubik belongs here as the bounded permutation model, not as a replacement for the larger ontology.
Onion belongs here as the safety-shell model.
The core remains live truth.
